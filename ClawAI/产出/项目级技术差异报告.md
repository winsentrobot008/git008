### 2.2 ⚠️ 本地任务卡死的根因分析（已证实）

| 问题点 | Source A (HF) | Source B (Fork) | 本地 ClawAI | 影响评估 |
|--------|---------------|----------------|---------------|----------|
| **后台任务管理** | `BackgroundTasks.add_task()` 框架级 | N/A（无 API 提交） | `asyncio.create_task()` fire-and-forget | ⚠️ 本地任务可能被静默回收 |
| **API 调用** | server.py → call_deepseek() (2跳) | CLI 直接调用 (1跳) | 5层代理链 | ⚠️ 任一层超时/报错都表现为无响应 |
| **LiveAgent 初始化** | 不存在 | CLI 中初始化 | 每次任务初始化 | ⚠️ 初始化失败则整个任务卡死 |
| **Base URL 配置** | `https://api.deepseek.com/v1` 硬编码 | LiveAgent 内部 | LiveAgent 内部 | ⚠️ 不匹配则 API 调用全失败 |
| **scheduler 存在性** | ✅ 有 | ❌ **没有** | ✅ 有 | 本地架构更接近 HF，但实现不同 |
| **故障隔离** | 无（直接报错） | CLI 异常终止 | Mock 三层回退 | ⚠️ Mock 可能吞掉真实错误 |

**已证实的根因**:

**根因 1** — Source B (Fork) 本身就是**没有** `task_scheduler.py` 的简化版本。如果你期望本地代码"应该和 Source B 一样能运行"，但实际上本地有 `task_scheduler.py` 而 Source B 没有，这说明本地代码是从 **HF 版**（或某个中间版本）衍生而来，而非直接基于 Source B。

**根因 2** — `asyncio.create_task()` vs `BackgroundTasks.add_task()`:
- HF 版使用 `BackgroundTasks`（FastAPI 框架级管理），**请求结束后任务仍然存活**
- 本地版使用 `asyncio.create_task()`，没有持有引用，可能被事件循环回收
- 本地版 `server.py:752` 直接调用 `scheduler.submit_task()` 是 `async` 方法，但内部 `asyncio.create_task(self._execute_task(...))` 在方法返回时没有保存 task 引用

**根因 3** — Base URL 不匹配:
- HF 版: `DEEPSEEK_API_BASE = os.getenv("DEEPSEEK_API_BASE", "https://api.deepseek.com/v1")`（server.py 第 837 行）
- 本地版: 通过 LiveAgent 间接读取，**可能使用默认 OpenAI Base URL `https://api.openai.com/v1`**
- 如果 `.env` 中只有 `DEEPSEEK_API_KEY` 而没有 `OPENAI_API_KEY`，发送到 OpenAI 的 URL 会返回 401

## 五、结论与修复建议

### 5.1 最终结论

1. **Source B (Fork 仓库) 是原始上游代码**，**没有** `POST /api/tasks` 端点，**没有** `task_scheduler.py`，Agent 通过 CLI 批量运行
2. **Source A (HF 线上版) 是你深度定制的生产版本**，用 `BackgroundTasks + call_deepseek()` 直调 DeepSeek，支持 Web 任务提交
3. **本地 ClawAI 是两者的混合**：端点架构类似 Source B，任务调度类似 Source A 但使用不同实现（`asyncio.create_task` vs `BackgroundTasks`）
4. **任务卡死根因**: `asyncio.create_task()` 的 fire-and-forget 语义 + `LiveAgent` 初始化阻塞 + Base URL 不匹配

### 5.2 推荐修复方案

**第一步：修复任务生命周期**
```python
# server.py — 将 submit_task 改为使用 BackgroundTasks
from fastapi import BackgroundTasks  # 添加导入

@app.post("/api/tasks", response_model=TaskSubmitResponse)
async def submit_task(request: TaskSubmitRequest, background_tasks: BackgroundTasks):
    scheduler = _get_scheduler()
    task_id = f"task_{uuid.uuid4().hex[:12]}"
    # 使用 BackgroundTasks 替代 asyncio.create_task
    background_tasks.add_task(scheduler._execute_task, task_id, ...)
    return TaskSubmitResponse(task_id=task_id, status="queued", ...)
```

**第二步：验证 DeepSeek Base URL**
```python
# server.py — 添加显式配置（参考 HF 版）
DEEPSEEK_API_BASE = os.getenv("DEEPSEEK_API_BASE", "https://api.deepseek.com/v1")
DEEPSEEK_API_KEY = os.getenv("DEEPSEEK_API_KEY", os.getenv("OPENAI_API_KEY", ""))
```

**第三步：启动时预热调度器**
```python
@app.on_event("startup")
async def startup_event():
    asyncio.create_task(watch_agent_files())
    # 预热调度器
    scheduler = _get_scheduler()
    print(f"✅ 调度器已预热: {scheduler}")
```

**第四步（可选）：添加直调 fallback**
参考 HF 版 `call_deepseek()` 方法，增加不经过 LiveAgent 的直接调用路径，提高可靠性。
